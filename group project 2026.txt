Datamak Technologies require you to design and develop a Fraud Detection and Risk  Monitoring System using the Neo4j database (graph database). The system will model  relationships between entities such as Customers, Transactions, Devices, Locations, and  Accounts to detect suspicious patterns and high-risk behaviour. Students must design a graph  schema with appropriate nodes, relationships, and properties, and implement Cypher queries to  analyse transaction flows, identify unusual connections, and compute risk levels. The system  should automatically flag high-risk transactions based on relationship patterns such as multiple  linked accounts, shared devices, high-frequency transfers, or connections to previously flagged  entities. 
This project will enable you to apply graph database concepts including graph modelling,  relationship traversal, pattern matching, and query optimization using Cypher. Groups must  submit a documented graph design (ER-to-graph mapping), sample dataset, implemented  queries, and a functional prototype demonstrating fraud detection scenarios. 
You must also write a report, and it must be in this format, “Abstract, Introduction, “Body”,  Conclusion, and References”. The report must include more than ten (10) academic  references published within the last five (5) years, sourced strictly from peer-reviewed  journals and academic books. References from blogs, non-academic websites, or  informal online sources are not permitted and IEEE referencing format must be followed.
